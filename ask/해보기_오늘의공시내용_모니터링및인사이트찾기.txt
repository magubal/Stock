그 플랜과 설계를 내가 claude code에게 넘겨서, bkit으로 다시 상세설계해서 처리하게 하려고 해...

---------------------

1) 웹페이지에서 꼭 뽑아야 하는 정리 항목(=인사이트가 생기는 구조)
A. 원문 공시에서 “그대로” 가져오는 기본 필드 [확인]

일자/시간, 시장(유가/코스닥/코넥스/파생), 회사명 검색(자동완성), 엑셀/ RSS 제공 같은 탐색 기능은 원본 페이지에도 이미 존재하므로 그대로 살리는 게 좋습니다.

B. 투자심리를 바꾸는 “사건 타입(이벤트 택소노미)” [추정→확인 필요]

공시는 제목만으로도 대략 분류가 됩니다. 핵심은 **“주가에 바로 영향”**과 **“리스크의 구조 변화”**를 갈라내는 겁니다.

긍정(리스크온/프리미엄 확대) 계열

자사주 매입/소각, 배당/주주환원, 대형 수주, 실적 서프/가이던스 상향, 전략적 제휴(MOU 중에서도 계약·금액 확정)

부정(리스크오프/디스카운트 확대) 계열

유상증자/CB·BW 등 희석성 자금조달, 실적 쇼크/가이던스 하향, 영업정지·소송·규제, 감사의견/내부회계 이슈, 최대주주 변경/담보제공(지배구조 불안)

중립(정보 갱신) 계열

단순 정정/단순 보고, 경미한 임원변동, 루틴 공시(단, “반복 빈도”가 급증하면 심리 신호가 됨)

분류 근거를 확실히 하려면, KIND의 “상세검색”에 존재하는 공시유형 체계(수시/정기/지분/발행/시장조치 등)를 기준축으로 가져오는 게 깔끔합니다.

C. “숫자”가 들어가면 인사이트가 생긴다: 핵심 수치 추출 [추정]

이벤트별로 1~3개 숫자만 뽑아도 체감이 달라집니다.

수주: 계약금액 / 매출 대비 / 기간

실적: 매출/영업이익 변동률, 흑자전환 여부

자금조달: 규모, 할인율, 전환가·리픽싱, 사용처

주주환원: 금액, 소각 여부, 기간

지분/지배구조: 변동 주체, 지분율 변화, 담보비율

D. “투자심리 변화”를 읽는 5개 파생 지표 [추정]

원문을 요약하는 게 아니라, 하루의 공시 흐름을 ‘심리’로 번역합니다.

Risk-on vs Risk-off 비중: 긍정/부정 이벤트 카운트와 “가중치 합”

희석(자금조달) 압력 지수: 유증/CB/BW 규모 합(시총 대비)

주주환원 온도: 자사주·배당 관련 규모/빈도

리스크 경보: 거래정지·실질심사·불성실공시·횡령 등 “시장조치/유의사항” 연결

클러스터(테마) 감지: 같은 키워드(예: “단일판매공급계약”, “자기주식취득”)가 특정 섹터에 동시다발 → “시장 서사가 만들어지는 중” 신호

2) 화면 설계(UX): “오늘의 공시”를 ‘투심 대시보드’로 바꾸기
0) 상단 고정 바 (탐색/필터) [확인]

날짜 선택, 시장 선택(전체/유가/코스닥/코넥스/파생), 회사명 검색(자동완성), Excel/RSS 내보내기
→ 원본 UI를 “유지”하되, 아래 인사이트 모듈과 연결.

1) 1열: 오늘의 “핵심 변화” 한 줄 요약 (데일리 브리프)

예: “오늘 공시는 자금조달(희석) 증가 + 주주환원 감소로, 단기 투심은 방어적 성격”

옆에 작은 배지: [확인] 원문 기반 / [추정] 해석

2) 2열: KPI 카드 6개 (숫자로 분위기 잡기)

Risk-on 이벤트 수 / Risk-off 이벤트 수

희석성 자금조달 규모(시총 대비)

주주환원 규모

‘정정/번복/해지’ 카운트(신뢰도 흔들림)

리스크 경보(거래정지·실질심사 등) 카운트

“집중도”: 상위 10개 이벤트가 전체 영향의 몇 %인지(특정 이슈 쏠림)

3) 메인 피드: “공시 카드” 리스트 (여기가 본체)

각 공시를 행(row)이 아니라 카드로:

(좌) 회사명/시장/시간

(중) 공시 제목 + 자동 분류 태그(수주/실적/조달/환원/지배구조/리스크…)

(우) 임팩트 점수(0~100) + 방향(+, -, ±) + “근거 2줄”

펼치면: 핵심 숫자 3개, 원문 링크, 관련 과거 공시(최근 90일) 스냅샷

4) 우측 사이드 패널: “클러스터 & 심리 변곡”

오늘 급증 키워드 TOP 10

섹터별로 어떤 타입이 많은지(조달↑ / 환원↑ / 수주↑)

“변곡 후보” 알림: 정정/해지가 많은 날, 조달이 동시다발인 날 등

3) 실행계획(데이터 → 분류/요약 → 검증 → 운영)
Phase 1) MVP (1~2주짜리로도 가능) — “분류 + 중요도 랭킹”부터

수집

우선은 KIND 페이지가 제공하는 Excel/RSS 흐름을 활용(가능한 범위에서)

만약 웹이 내부적으로 POST 호출로 목록을 받는 구조라면, 개발자도구 Network에서 실제 엔드포인트/파라미터를 확인해 정식화(사이트 정책 준수 전제). (과거 사례로 POST 기반 언급이 있으나, 현재도 동일한지는 반드시 재검증 권장)

규칙 기반 분류(1차)

공시 제목 키워드 매핑으로 이벤트 타입/방향(+/-) 추정

중요도 점수(1차)

이벤트 타입 가중치(예: 거래정지>유증>실적>수주>일반)

“정정/해지/번복” 가중치 추가(신뢰도 하락 시그널)

UI

상단 필터 + KPI 카드 + 공시 카드 리스트 + 상세 패널

Phase 2) V1 — “숫자 추출 + 시총 대비”로 임팩트 정교화

공시 본문/첨부에서 금액·비율·기간 정규표현식 추출

외부 데이터(시총/업종) 붙여서 상대적 크기(시총 대비, 매출 대비) 계산

“같은 회사 반복 공시”를 묶어 스토리라인(연속 이벤트) 제공

Phase 3) V2 — “투심 변화”를 진짜로 만들기(클러스터/내러티브)

키워드/섹터 클러스터링으로 “오늘 시장이 무슨 이야기를 만들고 있는지” 자동 요약

이벤트가 장중/장후에 나온 시간대와 다음날 갭/거래대금 반응을 결합(가능하면)

운영·검증 체크리스트(중요)

[확인] 공시 원문/목록은 KIND가 1차 출처로 고정

[확인] 수집 방식이 RSS/Excel/내부 API 중 무엇이 안정적인지 테스트

[추정→확인] “중요도 점수”는 처음엔 규칙 기반 → 누적 데이터로 캘리브레이션

[확인] 법/정책(robots, 이용약관) 준수 범위에서만 자동화
