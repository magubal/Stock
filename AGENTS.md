# Stock 프로젝트 - AI 에이전트 컨텍스트

## 프로젝트 개요

이 프로젝트는 **마구티어 투자 플라이휠** 시스템으로, 데이터 기반의 체계적인 투자 의사결정을 지원합니다.

## 투자 철학

📄 **상세 문서**: [investment-philosophy.md](./docs/investment-philosophy.md)

### 핵심 원칙
- **시장방향성 및 투자심리 이해** → 행동 가능성 예측
- **미래 최고선호기업 축적** 및 운영
- **7단계 플라이휠 사이클** 반복을 통한 지속적 개선

---

## ⚠️ **업무 연속성 정책 (모든 모델 필수)**

### 세션 시작 시 필수 절차
1. `AGENTS.md` 읽기 (이 파일)
2. `REQUESTS.md` 확인 (요청/요건 파악)
3. 개발 로그 확인 (`docs/development_log_*.md`)
4. 업무 연속성 확보 후 작업 시작

### 📋 요청 관리 정책

> **모든 AI 모델은 새로운 요청을 받으면 REQUESTS.md에 등록해야 합니다.**

#### 요청 등록 형식
```markdown
### REQ-XXX: [요청 제목]
| 항목 | 내용 |
|------|------|
| **요청 ID** | REQ-XXX |
| **상태** | 대기/진행/완료 |

#### 요구사항
1. [요건 1]
2. [요건 2]

#### 관련 파일
| 파일 | 위치 | 용도 |
|------|------|------|
```

#### 필수 기록
- **설정 파일 위치**: 리스트, 설정 등 외부 파일 경로
- **관련 스크립트**: 처리 프로그램 목록
- **데이터 저장 위치**: 결과물 폴더

### 🔄 지속적 개선 정책

> TODO가 없을 때 `investment-philosophy.md` 기반 시스템 보완점 분석 및 제안

## 워크플로우 (7단계 플라이휠)

| 슬래시 명령어 | 단계 | 핵심 활동 |
|-------------|------|----------|
| `/01-data-collection` | 1. 데이터 수집 | 뉴스, 블로그, 텔레그램, 보고서 자동 수집 |
| `/02-context-analysis` | 2. 맥락연결/영향분석 | 투자자심리 → 가능행동 유형 분석 |
| `/03-importance-evaluation` | 3. 중요도 파악 | 시장/섹터/종목 영향 및 방향성 추정 |
| `/04-decision-scenario` | 4. 의사결정 시나리오 | 행동 시나리오 및 우선순위 정리 |
| `/05-execution-check` | 5. 실질확인 | 시세/실제행동 확인 및 대응 |
| `/06-review-improvement` | 6. 결과확인/복기 | 복기 및 개선점 도출 |
| `/07-trend-research` | 7. 트렌드 핵심정리 | 독서/리서치 인사이트 정리 |

**워크플로우 파일 위치**: `.agent/workflows/`

## 스킬

| 스킬 | 설명 | 경로 |
|-----|------|-----|
| **Analysis** | 맥락분석, 영향력 평가, 센티먼트 분석 | `.agent/skills/analysis/` |
| **Data Collection** | 뉴스, 보고서, 텔레그램 데이터 수집 | `.agent/skills/data-collection/` |
| **Decision** | 의사결정 지원, 시나리오 작성, 리스크 평가 | `.agent/skills/decision/` |
| **Research** | 기업분석, 트렌드 리서치, 인사이트 정리 | `.agent/skills/research/` |

## 프로젝트 구조

```
Stock/                         ← 프로덕션 (안정 버전)
├── AGENTS.md                  # 핵심 정책 (이 파일)
├── docs/
│   └── investment-philosophy.md
├── .agent/
│   ├── workflows/             # 검증된 워크플로우
│   └── skills/                # 검증된 스킬
├── Test_02/                   ← 개발/테스트 환경 ⭐
│   ├── AGENTS.md              # 상세 정책 (실험 중)
│   ├── REQUESTS.md            # 요청 관리
│   └── TODO.md                # 작업 목록
└── test_data/                 # 공용 테스트 데이터
```

### 🔄 승격 프로세스 (Test_02 → Stock)

1. **Test_02에서 개발/검증** - 새 정책, 스킬, 워크플로우 테스트
2. **완성도 확인** - 충분히 안정화되면
3. **Stock에 머지** - 검증된 것만 프로덕션에 적용

## 사용 가이드

### 워크플로우 실행
슬래시 명령어로 각 단계를 실행합니다: `/01-data-collection`

### 투자 판단 시 참고
1. **공감 관점**: 이슈의 단기/중장기 영향 분석
2. **시간 관점**: 3개월/6개월/1년 기간별 평가
3. **고객 관점**: 중장기 투자자에게 매도할 수 있는 종목 선별
