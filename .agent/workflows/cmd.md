# /cmd 터미널 가디언 워크플로우 (Terminal Guardian)

사용자가 `/cmd [명령어]` 를 호출하거나, 터미널 실행이 요구되는 복잡한 상황일 때 AI는 백그라운드 방치 및 무한 로딩 실수를 방지하기 위해 다음 스텝을 **반드시 순차적으로** 수행해야 합니다.

1. **실행 환경 검토**: 사용자가 요청한 명령어가 무한 대기가 발생할 수 있는 파이프(`|`) 연결이나 네트워크 요청(`curl`)인지 확인합니다.
2. **명령어 실행**: `run_command` 도구를 사용하여 명령어를 실행합니다. 서버 무한 실행 목적이 아니라면 `WaitMsBeforeAsync` 파라미터를 최소 5000ms 이상 넉넉히 주어 동기적으로 끝나도록 유도합니다.
3. **상태 즉시 검증 (가장 중요)**: 
   - 절대 다음 코드 수정 단계로 혼자 넘어가지 마십시오.
   - 명령어가 백그라운드로 넘어갔다면 (CommandId가 반환되었다면), **반드시** 즉각적으로 `command_status` 도구를 호출하여 출력 결과나 에러를 확인합니다.
4. **결과 보고 및 중단**: 성공, 실패, 혹은 타임아웃 상태를 확인한 후 사용자에게 명확히 보고합니다. 에러가 났을 경우 절대 독단적으로 코드를 고치지 말고 진행을 멈춘 뒤 사용자의 지시를 대기합니다.
