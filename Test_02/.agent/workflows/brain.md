사용자가 `/brain <주제>` 를 실행하면 아래 순서를 따릅니다.

## 실행 순서

### 1. brainstorm 스킬 적용

`.agent/skills/brainstorm-bkit/SKILL.md`의 프로토콜을 **그대로** 실행합니다:

1. **Step 0 — 문제 정의**: 사용자의 주제를 1문장으로 재정의 + 성공 기준(DoD) 제안
2. **Step 1 — 제약/맥락 수집**: 빠진 맥락을 1~3개 질문으로 확인 (AskUserQuestion 사용)
3. **Step 2 — 대안 생성(발산)**: 2~4개 대안, 각각 핵심/장점/단점/실패모드
4. **Step 3 — 비교/선택(수렴)**: 기준 4~6개 비교표 → 추천안 1개 + 뒤집히는 조건

### 2. 결과 포맷 출력

SKILL.md의 **출력 포맷**을 정확히 따라 Design Brief를 생성합니다:
- 목표/성공 기준
- 전제/제약
- 대안(2~4개)
- 비교표
- 추천안 + 이유 + 뒤집히는 조건
- 리스크/완화 Top 3
- bkit handoff 섹션

### 3. PDCA plan handoff

결과 출력 후 마지막에 반드시:
- "다음 단계: `/pdca plan <feature-name>`" 안내
- plan 문서에 넣을 핵심 포인트 bullet 5~10개 정리
- 사용자가 바로 `/pdca plan`을 실행할 수 있도록 feature name 제안

## 주의사항
- 이 워크플로우에서는 **코드를 작성하지 않는다**. 의사결정만 한다.
- 대안은 최소 2개. "한 가지 답" 강요 금지.
- 질문은 한 번에 1~3개만 (사용자 피로 방지).
- 결론에는 반드시 "리스크 + 완화책"을 포함한다.
