## 1. 시스템 개요

### 1.1 배경 및 목적

투자자는 매일 다양한 시장 분석 작업을 수행하며 아이디어를 축적한다. 현재 이 작업들은 ChatGPT, Claude, Gemini 등 여러 AI 세션에 분산되어 있어:

- **아이디어 유실**: 세션 종료 시 맥락과 인사이트가 사라짐
- **연결 부재**: 서로 다른 AI에서 도출한 분석이 연결되지 않음
- **체계 부재**: 일일 작업 데이터가 단순 기록에 머물고, 아이디어로 발전하지 못함
- **협업 불가**: 각 AI 세션이 독립적으로 작동하여 시너지 없음

본 시스템은 이 문제들을 해결하여 **아이디어의 체계적 관리 + 다중 AI 간 협업**을 가능하게 한다.

### 1.2 핵심 가치

1. **아이디어 생명주기 관리**: 일일 작업 → 인사이트 추출 → 아이디어 생성 → 평가 → 실행 → 결과 추적
2. **크로스-AI 협업**: Claude, Gemini, ChatGPT 간 분석 맥락과 결론을 공유하는 표준화된 프로토콜
3. **투자 의사결정 지원**: 분산된 분석을 통합하여 actionable한 투자 판단 근거 제공

## 2. 일일 작업 데이터 구조 (입력 데이터)

### 2.1 데이터 소스

- **파일**: `data/ask/Idea_what can I do with dailyWork data for better stock system.xlsx`
- **형식**: Excel (.xlsx), 단일 시트

### 2.2 7개 작업 카테고리

| # | 카테고리 | 약칭 | 설명 |
|---|---------|------|------|
| 1 | 시장섹터선호변화 | SECTOR | 4분면 분석(Market Sector/Style Preference Quadrant) 기반 섹터 로테이션 추적 |
| 2 | 미국시장이슈및동향 | US_MARKET | 미국 시장 핵심 이슈, 거시경제 영향, 내러티브 분석 |
| 3 | 시장테마흐름 | THEME | 시장 테마의 발생·성장·소멸 추적, 테마 간 연결 분석 |
| 4 | 주식시장리스크 및 현금비중 전략 | RISK | VIX, 신용스프레드, 시장 리스크 평가 + 현금비중 전략 |
| 5 | 익일시장추정및대응 | NEXT_DAY | 다음 거래일 시장 방향 예측 및 대응 전략 |
| 6 | 투자시장 및 보유종목 관련 대응및생각 | PORTFOLIO | 보유 종목별 대응 전략, 매수/매도 판단 근거 |
| 7 | 투자시장 AI 리서치 및 생각 | AI_RESEARCH | AI를 활용한 투자 심리, 내러티브, 메타인지 연구 |

### 2.3 데이터 컬럼 구조

| 컬럼 | 타입 | 설명 |
|------|------|------|
| 날짜 | Date | 작업 수행일 |

## 3. 기능 요구사항

### 3.1 아이디어 매니저 에이전트 (FR-IDEA)

아이디어의 전체 생명주기를 관리하는 핵심 에이전트.

#### FR-IDEA-01: 일일 작업 데이터 수집 (Ingestion)

| 항목 | 내용 |
|------|------|
| **기능** | Excel 파일에서 일일 작업 데이터를 파싱하여 DB에 저장 |
| **입력** | `.xlsx` 파일 경로 |
| **처리** | 1) 시트 읽기 → 2) 카테고리 분류 → 3) 날짜별 그룹핑 → 4) DB INSERT |
| **출력** | 수집된 레코드 수, 카테고리별 통계 |
| **제약** | 한국어 텍스트 인코딩 처리, 중복 데이터 방지 (날짜+카테고리 unique) |

#### FR-IDEA-02: 인사이트 자동 추출

| 항목 | 내용 |
|------|------|
| **기능** | 상세내용에서 핵심 인사이트(주장, 예측, 패턴)를 자동 추출 |
| **처리** | LLM 기반 분석: 1) 핵심 주장 추출 → 2) 예측/가설 식별 → 3) 키워드·엔티티 태깅 |
| **출력** | 인사이트 목록 (type: claim/prediction/pattern, confidence, keywords) |
| **예시** | "섹터 로테이션이 방어주→성장주로 전환 중" → `{type: "pattern", keywords: ["섹터로테이션", "방어주", "성장주"]}` |

#### FR-IDEA-03: 아이디어 생성 및 등록

| 항목 | 내용 |
|------|------|
| **기능** | 추출된 인사이트를 기반으로 투자 아이디어 생성 |
| **구조** | `{id, title, category, thesis, evidence[], status, priority, created, updated, outcome}` |
| **상태 흐름** | `draft` → `active` → `testing` → `validated` / `invalidated` → `archived` |
| **우선순위** | P1(즉시실행) ~ P5(참고), LLM + 규칙 기반 자동 산정 |
| **연결** | 관련 인사이트, 관련 아이디어, 관련 AI 세션 링크 |

#### FR-IDEA-04: 아이디어 평가 및 검증 추적

| 항목 | 내용 |
|------|------|
| **기능** | 아이디어의 유효성을 시장 결과와 대조하여 평가 |
| **처리** | 1) 예측 vs 실제 비교 → 2) 정확도 산정 → 3) 패턴 학습 |
| **지표** | hit_rate, avg_return, risk_adjusted_score |
| **알림** | 검증 시점 도래 시 사용자에게 리마인더 |

#### FR-IDEA-05: 아이디어 연결 그래프

| 항목 | 내용 |
|------|------|
| **기능** | 아이디어 간 관계를 그래프로 시각화 |
| **관계 유형** | supports(지지), contradicts(모순), extends(확장), depends_on(의존) |
| **시각화** | 네트워크 그래프 (Force-directed), 카테고리별 색상 구분 |
| **활용** | 모순되는 아이디어 발견, 강화되는 테마 식별 |

#### FR-IDEA-06: 카테고리별 트렌드 대시보드

| 항목 | 내용 |
|------|------|
| **기능** | 7개 카테고리별 시계열 트렌드 및 요약 |
| **차트** | 카테고리별 아이디어 빈도, 인사이트 히트맵, 키워드 클라우드 |
| **필터** | 날짜 범위, 카테고리, 상태, 우선순위 |

### 3.2 크로스-AI 협업 허브 (FR-COLLAB)

여러 AI 세션 간 맥락과 분석 결과를 공유하는 시스템.

#### FR-COLLAB-01: AI 세션 컨텍스트 패킷 (Context Packet)

| 항목 | 내용 |
|------|------|
| **기능** | AI 세션의 맥락을 표준화된 JSON 포맷으로 내보내기/가져오기 |
| **패킷 구조** | 아래 참조 |
| **용도** | Claude에서 분석한 내용을 Gemini에 전달, 또는 그 반대 |

```json
{
  "packet_version": "1.0",
  "source_ai": "claude",
  "session_id": "uuid",
  "timestamp": "2026-02-14T10:00:00Z",
  "topic": "섹터 로테이션 분석",
  "category": "SECTOR",
  "context": {
    "background": "현재 시장 상황 요약...",
    "analysis": "분석 내용...",
    "conclusions": ["결론1", "결론2"],
    "open_questions": ["미해결 질문1"],
    "data_references": ["참조 데이터 경로1"]
  },
  "request_to_next_ai": {
    "action": "validate|extend|challenge|synthesize",
    "specific_ask": "이 분석의 반론을 검토해주세요",
    "priority": "high"
  },
  "metadata": {
    "related_ideas": ["IDEA-001"],
    "confidence": 0.7,
    "tags": ["섹터로테이션", "방어주"]
  }
}
```

#### FR-COLLAB-02: 협업 대화 스킬 (Collaboration Dialogue Skill)

| 항목 | 내용 |
|------|------|
| **기능** | AI 간 협업을 위한 구조화된 대화 흐름 제공 |
| **스킬 이름** | `/collab` |
| **서브커맨드** | 아래 참조 |

| 커맨드 | 기능 | 예시 |
|--------|------|------|
| `/collab export` | 현재 세션 맥락을 Context Packet으로 내보내기 | `/collab export --topic "섹터분석"` |
| `/collab import` | 다른 AI의 Context Packet 가져오기 | `/collab import data/collab/packet_001.json` |
| `/collab review` | 가져온 패킷의 분석을 검토하고 의견 추가 | `/collab review --mode challenge` |
| `/collab synthesize` | 여러 AI 분석을 종합하여 통합 결론 도출 | `/collab synthesize --packets p1,p2,p3` |
| `/collab status` | 현재 협업 현황 (교환된 패킷, 진행 중 토론) | `/collab status` |

#### FR-COLLAB-03: 멀티-AI 세션 레지스트리

| 항목 | 내용 |
|------|------|
| **기능** | 활성 AI 세션들의 목록 및 작업 현황 관리 |
| **등록 정보** | AI 종류, 세션 ID/링크, 할당 작업, 상태, 마지막 교류 시각 |
| **활용** | 어떤 AI가 어떤 분석을 담당하고 있는지 한눈에 파악 |

#### FR-COLLAB-04: 협업 워크플로우 템플릿

| 항목 | 내용 |
|------|------|
| **기능** | 자주 사용하는 AI 협업 패턴을 템플릿으로 정의 |
| **템플릿 예시** | 아래 참조 |

| 워크플로우 | 패턴 | 설명 |
|-----------|------|------|
| 교차 검증 | Claude분석 → Gemini검증 → Claude종합 | 한 AI의 분석을 다른 AI가 검증 |
| 다관점 분석 | 동시(Claude+Gemini+ChatGPT) → 종합 | 동일 주제를 여러 AI가 병렬 분석 |
| 심화 연구 | Claude초안 → Gemini심화 → ChatGPT시각화 | 단계적 심화 분석 |
| 반론 토론 | Claude(찬성) ↔ Gemini(반대) → 종합 | 의도적 반론 생성 및 토론 |

#### FR-COLLAB-05: 협업 히스토리 및 결과 추적

| 항목 | 내용 |
|------|------|
| **기능** | AI 간 교환된 분석과 그 결과를 타임라인으로 기록 |
| **저장** | 패킷 교환 이력, 각 AI 의견, 최종 종합 결론, 실제 결과 |
| **학습** | 어떤 협업 패턴이 좋은 결과를 냈는지 통계 분석 |

### 3.3 도메인 전문 에이전트 (FR-AGENT)

7개 카테고리에 대응하는 전문 분석 에이전트.

#### FR-AGENT-01: 섹터 로테이션 에이전트 (SECTOR)

| 항목 | 내용 |
|------|------|
| **기능** | 시장 섹터/스타일 선호 변화 추적 및 분석 |
| **입력** | 4분면 분석 데이터 (Growth/Value × Large/Small) |
| **분석** | 로테이션 방향 감지, 전환 시그널, 과거 패턴 비교 |
| **출력** | 현재 선호 섹터, 변화 속도, 확신도, 추천 포지셔닝 |

#### FR-AGENT-02: 미국시장 동향 에이전트 (US_MARKET)

| 항목 | 내용 |
|------|------|
| **기능** | 미국 시장 핵심 이슈 정리 및 거시 영향 분석 |
| **입력** | 미국 시장 관련 뉴스, 경제지표, 내러티브 |
| **분석** | 이슈 중요도 평가, 한국 시장 영향 경로 분석, 내러티브 변화 추적 |
| **출력** | 핵심 이슈 요약, 영향도 매트릭스, 주의 시그널 |

#### FR-AGENT-03: 테마 흐름 에이전트 (THEME)

| 항목 | 내용 |
|------|------|
| **기능** | 시장 테마의 생명주기(발생→성장→피크→쇠퇴) 추적 |
| **입력** | 테마 관련 데이터 (언급 빈도, 관련 종목 움직임) |
| **분석** | 테마 단계 판별, 연관 테마 네트워크, 모멘텀 측정 |
| **출력** | 테마별 단계, 강도 점수, 연관 종목 리스트 |

#### FR-AGENT-04: 리스크 관리 에이전트 (RISK)

| 항목 | 내용 |

| **분석** | 리스크 레벨 산정, 적정 현금비중 산출, 헤지 전략 제안 |
| **연동** | `liquidity-stress-monitor`와 데이터 공유 |
| **출력** | 리스크 등급, 추천 현금비중, 경고 시그널 |

#### FR-AGENT-05: 익일 추정 에이전트 (NEXT_DAY)

| **검증** | 다음 날 실제 결과와 자동 대조 → 정확도 추적 |

#### FR-AGENT-06: 포트폴리오 대응 에이전트 (PORTFOLIO)

## 4. 비기능 요구사항

### 4.1 기술 아키텍처

| 항목 | 선택 | 근거 |
|------|------|------|
| **Frontend** | Static HTML + CDN React | 기존 `dashboard/` 패턴 유지 |
| **Backend** | FastAPI (기존 확장) | 기존 인프라 활용 |
| **DB** | SQLite (기존 확장) | 단일 사용자, 로컬 우선 |
| **AI 연동** | LLM API (OpenAI/Anthropic) | 인사이트 추출, 분석 |
| **파일 교환** | JSON 파일 + 클립보드 | AI 간 컨텍스트 전달 |

### 4.2 확장성

- 새로운 카테고리 추가 시 에이전트 확장 용이해야 함
- 새로운 AI 플랫폼 추가 시 Context Packet 포맷만 준수하면 연동 가능
- 향후 웹 기반 실시간 협업으로 확장 가능한 구조

### 4.3 데이터 보안

- 모든 데이터는 로컬에만 저장 (클라우드 전송 없음)
- API 키는 `.env`로 관리
- AI 세션 링크는 개인용 (외부 공유 불가)

### 4.4 사용성

- 한국어 UI 우선
- 기존 `dashboard/index.html` 시장모니터링 섹션에 자연스러운 링크 추가
- CLI 기반 스킬(`/collab`)은 직관적인 서브커맨드 구조

## 6. 크로스-AI 협업 프로토콜 상세

### 6.1 협업 흐름 예시

```
[사용자] "오늘 섹터 로테이션 분석해줘"
    ↓
[Claude] 분석 수행 → 결론 도출
    ↓
[사용자] "/collab export --topic 섹터로테이션"
    ↓
[시스템] Context Packet 생성 → data/collab/packets/에 저장
    ↓
[사용자] Gemini 세션에서 패킷 내용 전달 (복사/붙여넣기 또는 파일 업로드)
    ↓
[Gemini] 분석 검토 → 반론/보완 → 결과를 같은 포맷으로 출력
    ↓
[사용자] Gemini 결과를 Claude에서 "/collab import"
    ↓
[Claude] 양측 분석 종합 → 통합 결론 → 아이디어 생성/업데이트
```

### 6.2 Context Packet 교환 방식

현실적으로 AI 간 직접 통신은 불가능하므로, **사용자가 중개자** 역할:

| 방식 | 설명 | 장단점 |
|------|------|--------|
| **클립보드** | 패킷 JSON을 복사/붙여넣기 | 가장 간단, 대용량 불편 |
| **파일** | JSON 파일 저장 후 다른 AI에 업로드 | 구조화, 히스토리 보존 |
| **요약** | 핵심 내용만 자연어 요약으로 전달 | 정보 손실 있으나 편리 |

**추천**: 파일 기반 + 자연어 요약 병행 (상세 분석은 파일, 빠른 전달은 요약)

### 6.3 AI별 강점 활용 전략

| AI | 강점 | 추천 활용 |
|----|------|----------|
| **Claude** | 깊은 분석, 코드 작성, 구조화, 장기 맥락 | 핵심 분석, 시스템 구축, 종합 |
| **Gemini** | 최신 정보 검색, 다모달, Google 연동 | 실시간 뉴스 분석, 데이터 검증 |
| **ChatGPT** | 광범위 지식, 유연한 대화, 플러그인 | 아이디어 브레인스토밍, 시각화 |

## 7. 리스크 및 제약사항

| 리스크 | 영향 | 완화 방안 |
|--------|------|----------|
| AI 간 직접 통신 불가 | 협업에 사용자 개입 필수 | Context Packet 표준화로 전달 비용 최소화 |
| LLM 할루시네이션 | 인사이트 추출 정확도 저하 | confidence score + 사용자 확인 프로세스 |
| 데이터 양 증가 | SQLite 성능 한계 | 인덱싱 최적화, 아카이빙 정책 |
| 컨텍스트 윈도우 제한 | 대량 데이터 전달 불가 | 요약 + 핵심 데이터만 전달 |
