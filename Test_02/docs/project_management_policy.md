# 프로젝트 요청 관리 정책

## 🎯 **핵심 원칙**
1. **초기 요청 충실 준수** - 원래 요청을 변경하지 않음
2. **체계적 요청 추적** - 모든 요청/변경 기록
3. **일관된 파일 구조 유지** - 정해진 규칙 준수
4. **프로세스 표준화** - 반복 작업 자동화

## 📁 **데이터 수집 시스템 규격**

### **원래 요청 (고정)**
```
data/naver_blog_data/
├── 2026-02-01/
│   ├── daybyday_001.jpg (본문만)
│   ├── daybyday_002.jpg
│   ├── 라틴카페_001.jpg
│   └── 유수암바람_001.jpg
├── 2026-02-02/
│   └── ...
└── index/
    ├── request_history.json
    ├── change_log.json
    └── progress_status.json
```

## 🔄 **프로세스 표준**

### **요청 처리 프로세스**
1. **요청 기록** - request_history.json에 원본 요청 저장
2. **요구사항 분석** - 초기 요청과의 일관성 확인  
3. **변경 필요 시** - change_log.json에 기록 후 승인
4. **개발 진행** - 표준화된 프로세스 적용
5. **결과 검증** - 초기 요청 기준으로 검증
6. **진행 상황 업데이트** - progress_status.json 갱신

### **변경 관리 규칙**
- **초기 요청 우선** - 원래 요청을 최우선으로 존중
- **변경 시 명확한 사유** - 왜 변경이 필요한지 기록
- **롤백 준비** - 문제 발생 시 즉시 원복
- **일관성 유지** - 파일 구조, 명명 규칙 준수

## 🚀 **자동화 정책**

### **수집기 표준**
- **파일 구조**: 고정 `일자/블로거_순번.jpg`
- **품질 기준**: 정보 완성도 > 용량
- **실행 주기**: 매일 자동
- **검증**: 수집 즉시 품질 확인

### **코드 템플릿**
- **일관된 명명**: 파일명, 함수명 표준화
- **에러 처리**: 표준화된 예외 처리
- **로깅**: 진행 상황 상세 기록
- **테스트**: 모든 변경 사항 테스트 필수
